# Ciao! Site

The Ciao! marketing website — currently hand-maintained static HTML deployed straight to GitHub Pages, originally generated by an Astro + Vue-islands build whose source has been lost. This context covers the layout, page, and metadata vocabulary used when reasoning about its architecture.

## Language

**Page**:
A single route of the site (e.g. `/`, `/areas/web-design/`), authored as content + metadata and deployed as `<path>/index.html`.
_Avoid_: route, view, screen.

**Layout Shell**:
The single deep module every **Page** renders through — the head, sticky marquee banner, header/nav, cookie banner, and footer. Today it is copy-pasted across every Page (no seam); the deepening goal is to make it one module behind a small interface.
_Avoid_: template, partial, component, boilerplate.

**Metadata**:
The per-**Page** facts a Page must declare — title, description, path, social image — from which canonical, Open Graph, and Twitter tags are *derived* rather than hand-written.
_Avoid_: meta tags, SEO tags, head config.

**Animation-hooked zone**:
Markup whose exact classes and structure are read by `_hej/main.js` (the marquee, zoom-scroll sections, Lottie mounts) or by the contact **Island**. Preserved verbatim during migration; cleanup is not allowed here while `_hej` stays opaque.
_Avoid_: animated section, dynamic content.

**Inert zone**:
Markup `_hej` does not read — `<head>` **Metadata**, static body content, footer links/social. Safe to clean up freely during migration.

**Island**:
A hydrated interactive region embedded in an otherwise-static **Page**. Exactly one exists today: the ContactSteps multi-step form on `/contact/`, served from the opaque `_hej` bundle.
_Avoid_: widget, component, embed.

**`_hej`**:
The opaque build-output directory (a renamed Astro `_astro/`) holding the minified Vue/GSAP/Lottie bundles. Treated as a black box with no recoverable source; its inputs (the **Animation-hooked zone**) must not be refactored until its source is recovered.

## Relationships

- Every **Page** renders through exactly one **Layout Shell**.
- A **Page** declares **Metadata**; the **Layout Shell** derives canonical/OG/Twitter tags from it.
- A **Page** may contain an **Island**; an **Island** is served by `_hej`.
- The **Animation-hooked zone** of any **Page** is read by `_hej`; the **Inert zone** is not.

## Flagged ambiguities

- "component" was used loosely for both the **Layout Shell** and an **Island** — resolved: the Shell is the shared frame every Page renders through; an Island is a hydrated interactive region inside a Page.
- "boilerplate" described the duplicated Shell markup — resolved: it is not boilerplate but a **Layout Shell** that currently lacks a seam.
